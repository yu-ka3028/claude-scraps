# Claude Code Scraps

Claude Code のセッション中に出会った概念・用語を記録し、スペースドリピティション的に復習するための個人学習管理システム。CLAUDE.md と skillsだけで動きます。

---

## 概要

AI を使い始めてから、自分のスキルを超えた成果物が出るようになった。
生まれたギャップを広い範囲の学習で一気に埋めようとするより、その場で気になったことをひとつ拾って、忘れる前に繰り返す方が効率がいいのではと思い、Claude Code セッションの中に記録と復習を埋め込んでみました。

**課題:**
- TIL は書いて終わりで読み返さない
- 専用アプリへの入力はコンテキストスイッチが発生して続かなくなる

**解決策:**
Claude Code セッション中に「これ気になった」と思った瞬間に保存 → スペースドリピティションで定着まで追跡。

---

## ファイル構成

```
.
├── CLAUDE.md                        # ルール定義（Claudeへの指示）
├── _template.md                     # 新規カテゴリファイルのテンプレート
├── _archive.md                      # 理解度が完全に定着したスクラップの保管庫
├── {カテゴリ}.md                    # カテゴリ別スクラップ（SQL.md, TypeScript.md など）
└── .claude/
    └── skills/
        ├── status-scrap/skill.md    # /status-scrap コマンド
        └── review-scrap/skill.md   # /review-scrap コマンド
```

---

## 使い方

### 1. スクラップを保存する

セッション中の会話をテンプレート形式に整理して理解度を返答するだけで `git commit & push` まで実行する。

```
スクラップに保存して
```

**理解度レベル:**

| レベル | 意味 | 次回復習 |
|--------|------|----------|
| `[x][ ][ ][ ]` | 保存したばかり | 1日後 |
| `[x][x][ ][ ]` | 概要は言える | 3日後 |
| `[x][x][x][ ]` | 自分の言葉で説明できる | 7日後 |
| `[x][x][x][x]` | 完全に定着 | 即アーカイブ |

### 2. 今日の復習件数を確認する

```
/status-scrap
```

2026-02-28 のスクラップ状況
本日の復習対象: 5件（うち持ち越し 2件）

### 3. 復習セッションを開始する

```
/review-scrap
```

1問ずつ出題 → 回答 → フィードバック → 理解度申告 → ファイル更新、のサイクルを回す。
理解度 `[x][x][x][x]` を申告すると `_archive.md` に自動移動され、以降の復習には出題されない。

## スクラップのフォーマット

```markdown
## {トピック名}

| 項目 | 値 |
|------|----|
| 理解度 | [x][ ][ ][ ] |
| 保存日 | YYYY-MM-DD |
| 最終復習 | YYYY-MM-DD |
| 次回復習 | YYYY-MM-DD |

**Q:** {質問文}

**A:** {回答}

**メモ:** {補足・背景・使用例など（省略可）}
```

カテゴリが未分類であれば新規ファイルを作成、分類済みであれば追記されます。
