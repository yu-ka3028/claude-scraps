**Q:** スタックメモリとヒープメモリの違いは何か？

**A:** スタックは関数呼び出し・ローカル変数を自動管理する小さく高速な領域。ヒープは動的に確保する大きな領域でGCが管理する。プリミティブ型（数値・真偽値など）はスタック、オブジェクト・配列などの参照型はヒープに置かれる。

**メモ:**
- 再帰が深すぎるとスタックが溢れて `Maximum call stack size exceeded` が発生する
- イベントリスナーやクロージャがヒープのオブジェクトへの参照を持ち続けるとGCされずメモリリークになる
- bfcacheはヒープ状態ごとページを保存するため完全復元が可能。[[visibilitychange]] と組み合わせて理解する
