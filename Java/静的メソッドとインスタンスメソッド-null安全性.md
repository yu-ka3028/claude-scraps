**Q:** 静的メソッドとインスタンスメソッドで、null に対する振る舞いが異なるのはなぜ？

**A:** 静的メソッドは null を含む任意の値を「引数」として受け取れるので、内部で null チェックを実装できる。インスタンスメソッドは「存在するオブジェクト（レシーバー）」に対して呼ぶ設計のため、レシーバー自体が null だと JVM が操作対象なしと判断して NPE を投げる。

**メモ:** `StringUtils.hasText(str)`（静的）は null を渡しても false を返す。`str.isEmpty()`（インスタンス）は str が null だと NPE。同じ「空文字チェック」でも設計が違う。他にも `Objects.isNull(obj)` vs `obj.equals(null)` など同パターンが広く存在する。null を扱う場面では [[Java/int-Integer-null.md]] も参照。
