# JavaScriptの非同期処理 — async・await

JavaScriptはシングルスレッドで動いていますが、ネットワークリクエストやタイマーなど時間のかかる処理を非同期で扱える仕組みがあります。async/awaitはその仕組みをシンプルに書けるようにした構文です。

asyncをつけた関数の中でawaitを使うと、Promiseが解決するまでその行で待ってから次の行へ進みます。アプリ全体を止めるのではなく、その関数の中だけを一時停止するイメージです。.then()チェーンと比べて何がいいかというと、変数に値を直接受け取れるので見た目が同期処理と同じになります。上から順に読めるので、コードの意図がわかりやすいんですよね。

awaitはasync関数の中でしか使えません。エラーハンドリングはtry/catchで行って、awaitの行でPromiseが失敗するとcatchブロックに入ります。async/awaitはPromiseの糖衣構文なので内部ではPromiseが動いています。でも、Promiseを直接触る必要がなくなるのが大きなメリットで、非同期処理なのに同期的に書けるという体験が得られます。

非同期処理を理解するとAPIコールやファイル読み込みなど実用的な処理が書けるようになります。ReactのuseEffectでデータフェッチするときも、このasync/awaitを中で使うパターンが多いです。
