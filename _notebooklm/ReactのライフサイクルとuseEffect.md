# ReactのライフサイクルとuseEffect

Reactコンポーネントには生まれてから消えるまでのライフサイクルがあります。マウントはコンポーネントが画面に初めて表示されDOMに追加される瞬間、アンマウントはDOMから取り除かれる瞬間です。再レンダーはその間に何度も起きます。

useEffectはこのライフサイクルと副作用を結びつけるフックです。第2引数の依存配列でいつ実行するかを制御します。空の配列を渡すとマウント時のみ実行、変数を入れるとその値が変わったときに再実行、省略すると毎レンダリングで実行されます。

useEffectには対になるクリーンアップ関数があります。returnで関数を返すと、次にエフェクトが実行される直前かアンマウント時に呼ばれます。addEventListenerしたらremoveEventListener、接続したら切断、setTimeoutしたらclearTimeout、というようにペアで書くのが基本です。これをやらないと古いリスナーや接続がどんどん積み重なってメモリリークにつながります。

依存配列の値が変わるたびに「古いエフェクトのクリーンアップ → 新しいエフェクトの実行」というサイクルが回ります。つまりuseEffectの流れは、マウント時にエフェクト実行、依存配列変化時にクリーンアップしてから再実行、アンマウント時にクリーンアップ、という3段階で理解するとすっきりします。
