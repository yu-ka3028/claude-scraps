# Reactのフックを使いこなす — useStateの設計

ReactのフックはuseStateやuseEffectなどuseで始まる関数で、コンポーネントにロジックを持たせるための仕組みです。一つ重要なルールがあって、フックはループや条件分岐の中では使えません。理由は、Reactがフックを「N番目の呼び出し」という順序で管理しているからです。毎レンダリングで呼び出し回数が変わると管理が崩れます。解決策はループの中身を別コンポーネントに切り出すことで、各コンポーネントは常に同じ順序でフックを呼ぶようにします。

propsとstateの違いは、自分で変えられるかどうかです。propsは親から渡される値で読み取り専用、stateは自分がuseStateで管理する値で更新できます。

setStateで状態を更新するとき、前の値に依存するなら更新関数を使うのが正解です。setScore(prev => prev + 1)という形で、prevには確実に最新の値が渡されます。ゼロリセットや入力値の反映など前の値に依存しない場合は直接値を渡せばOKです。

初期値に重い計算を使いたい場合はuseState(heavyCalc)のように関数参照を渡します。heavyCalc()と書いてしまうと毎レンダリングで実行されますが、関数参照を渡すとReactが初回だけ呼んでくれます。「関数を渡して、呼び出しは受け取った側に任せる」というコールバックと同じ発想です。
