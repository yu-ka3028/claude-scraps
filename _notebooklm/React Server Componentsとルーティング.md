# React Server Componentsとルーティング

RSCはReact Server Componentsの略で、サーバー上でのみ動作するReactコンポーネントです。従来のSSRと何が違うかというと、JSをブラウザに送らないという点が大きいです。従来のSSRはHTMLをサーバーで生成した後、インタラクティブな操作のためにJavaScriptもブラウザへ送ってhydrationが必要でした。RSCはJavaScriptを送らないのでhydrationが不要で、バンドルサイズが減ります。データベースへの直接アクセスもできて、APIを経由する必要がありません。感覚としてはRails ERBに近くて、サーバーでHTMLを組み立てて返すイメージです。インタラクションが必要な部分だけuse clientをつけてクライアントコンポーネントにすることで、1ページの中でサーバーとクライアントを混在させられます。

Next.jsのApp RouterはこのRSCを活かすためのルーティング方式です。appフォルダ以下にファイルを置くだけでURLルートになるファイルベースルーティングを採用しています。Railsのconfig/routes.rbに明示的にルートを書くのとは真逆のアプローチで、ファイルの置き場所が規約になっています。ただ目的は同じで、URLとコードの対応付けです。デフォルトがサーバーコンポーネントになっているので、何もしなければRSCとして動き、必要な箇所だけクライアントコンポーネントに切り替えるという設計思想です。
