# SQLの書き方を整理する — CTEとJOINの本質

CTEはCommon Table Expressionの略で、WITHで始まる名前付きのサブクエリです。通常のサブクエリとの違いは、名前を持って後から複数回参照できる点と、複雑なクエリを上から順に整理して書けるという点です。ネストが深くなりがちなサブクエリを、「まずこのデータを準備して」「次にこれを準備して」と宣言的に書けるので、読みやすさが格段に上がります。Common Table Expressionの「テーブルエクスプレッション」という名前に、「名前をつけてテーブルのように扱える」という意味が込められています。

JOIN、特にINNER JOINについては、動作の本質を理解しておくと応用が効きます。実はFROM A, Bというカンマ区切りの書き方はCROSS JOIN、つまりAとBの全組み合わせの直積と同義なんです。そこにWHEREで絞り込むとINNER JOINと同じ結果になります。ただ、現代ではINNER JOIN ... ON ...と明示的に書く方が意図が伝わりやすいので推奨されています。カンマ区切りは「旧式のINNER JOIN記法」と説明されることもありますが、それは結果が同じというだけで、構文としてはクロス結合にフィルターをかけているという話です。JOINの本質を知っていると、実行計画を読むときや最適化を考えるときに役立ちます。
