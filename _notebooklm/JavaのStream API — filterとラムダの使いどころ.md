# JavaのStream API — filterとラムダの使いどころ

JavaのStream APIは、コレクションに対する操作を宣言的に書ける仕組みです。filterはStreamのチェーン内で要素を絞り込むメソッドで、ifとは役割が異なります。

Streamの中で「この要素を通すか除外するか」を決めるのがfilterの仕事です。条件に合う要素だけを後続の処理に流します。forEachやmapのブロック内にifを書いてスキップすることもできますが、それは絞り込みとは意味合いが違います。filterを使う方が「絞り込んでいる」という意図が明確に伝わるんですよね。

ifとの使い分けは単純で、Streamチェーン内で要素を絞り込むならfilter、単一の値に対する分岐や早期リターンにはifを使います。Streamの中でifを乱用するよりも、filterで先に絞り込んでからmapやforEachで処理する方がコードの流れが読みやすくなります。

注意点がひとつあって、ラムダ内で外部変数を参照する場合はeffectively final、つまり実質的に変更されない変数でないとコンパイルエラーになります。ラムダは後で実行される可能性があるため、参照している変数の値が変わると何が起きるかわからない、という理由から制約が設けられています。この制約はStreamに限らず、JavaのラムダやAnonymous classで共通のルールです。
