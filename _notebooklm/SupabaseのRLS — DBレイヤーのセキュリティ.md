# SupabaseのRLS — DBレイヤーのセキュリティ

RLSはRow Level Securityの略で、PostgreSQLが持つ行レベルのアクセス制御機能です。Supabaseで使うと何がいいかというと、クライアントからデータベースに直接アクセスしても安全にできる点です。

通常のWebアプリはExpressやRailsなどのAPIサーバーを経由してDBにアクセスします。このAPIサーバーでWHERE user_id = current_userのような絞り込みを書いて、他のユーザーのデータを見られないようにします。でもこれ、書き忘れたら即データ漏洩です。RLSはこの制御をDBレイヤーに強制します。デフォルトは全アクセス拒否で、明示的にポリシーを定義した操作だけを許可します。安全側に倒れているのがいいですよね。

SupabaseはPostgreSQLのRLSと認証機能のAuthがセットで統合されていて、ポリシーの中でcurrent_userが誰かを参照しながらアクセス制御を書けます。結果としてAPIサーバーを省略してクライアントからDBに直アクセスしても安全な設計が実現できます。インフラがシンプルになるのが大きなメリットで、バックエンドの薄いプロダクトや個人開発では特に恩恵が大きいです。

もちろんRLSだけで全部解決するわけではなく、複雑なビジネスロジックや集計処理にはAPIサーバーが必要になる場面もあります。ただ「認可はDBに任せる」という考え方は、書き忘れによるバグを構造的に防げるという点で設計上の強みになります。
