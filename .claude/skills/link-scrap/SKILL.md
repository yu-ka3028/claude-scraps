全スクラップを読み込み、関連するファイル間に `[[リンク]]` を追加・整理する定期メンテコマンド。

## 手順

1. `_index.md` を読み込み、全ファイルパスを取得する
2. 全スクラップファイルを読み込む
3. 内容をもとに関連性を判断する（キーワード・概念の重複、Q/Aの対比関係など）
4. 各ファイルのメモ欄を確認し、以下を行う:
   - 関連しているのにリンクがない箇所 → `[[ファイル名]]` を自然な文脈で追加
   - すでにリンクがある箇所 → スキップ（変更しない）
5. 変更したファイルのみ保存する
6. 変更内容のサマリーを表示する:

```
リンク更新:
  SQL/CTE.md → [[ウィンドウ関数-ページネーション最適化]] を追加
  Web/bfcache.md → リンクなし（変更なし）
  ...
```

7. コミット・プッシュする:

```bash
git add {変更したファイル}
git commit -m "scrap: update links (YYYY-MM-DD)"
git push origin main
```

## リンクを追加する判断基準

- 同じ概念・用語を別の角度から扱っている
- 一方が他方の前提知識になっている
- Q/Aで対比・比較されている関係にある
- メモ欄で「〜との違い」「〜も参照」のような参照が文章で書かれているが `[[]]` になっていない

## 注意

- メモ欄への追加は**自然な文脈**に限る。無理に埋め込まない
- 双方向にリンクを張る（A→B なら B→A も確認する）
- `_index.md` は変更しない（キーワード列はスクラップ保存時に管理する）
