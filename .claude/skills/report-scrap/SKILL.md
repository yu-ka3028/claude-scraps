期間を指定してスクラップの学習レポートを生成する。

## 手順

### 1. 期間を確認する

以下のメッセージを表示して期間を確認する:

```
期間を指定してください（省略時: month）:
  week   → 直近7日
  2weeks → 直近14日
  month  → 直近30日
```

ユーザーが省略した場合は `month` を使用する。

### 2. git log で活動を取得する

期間に応じて以下のコマンドを実行する:

```bash
git log --oneline --since="{N} days ago" --grep="^scrap:"
```

| 期間 | N |
|------|---|
| week | 7 |
| 2weeks | 14 |
| month | 30 |

コミットメッセージをパースして以下に分類する:
- `scrap: add ...` → 新規保存
- `scrap: review ...` → 復習セッション
- `scrap: archive ...` → アーカイブ（完全定着）

### 3. _index.md を読んで理解度の分布を集計する

現在のスクラップ全件について理解度を集計する:
- `[x][ ][ ][ ]`: N件
- `[x][x][ ][ ]`: N件
- `[x][x][x][ ]`: N件

### 4. _fundamentals.md と照合する

`_fundamentals.md` を読み込み、`_index.md` のキーワード列と照合して以下を判定する:

- **カバー済み**: キーワードがいずれかのスクラップに含まれている項目
- **未カバー**: スクラップが存在しない項目

### 5. 次の学習提案を3件ピックする

未カバー項目の中から、以下の優先順位で3件を選ぶ:
1. 現在のスクラップと同カテゴリの項目（関連して学びやすい）
2. 理解度の低いスクラップのキーワードに近い項目（補完できる）
3. 上記に該当しない場合は基礎度の高い項目から順に選ぶ

各提案には「なぜ今学ぶといいか」を1文で添える。

### 6. レポートを表示する

```
## {YYYY-MM-DD} 〜 {YYYY-MM-DD} のスクラップレポート

### 活動サマリー
- 新規保存: N件
  - {カテゴリ}: N件
- 復習セッション: N回
- 完全定着（アーカイブ）: N件

### 理解度の分布（現在）
- [x][ ][ ][ ] 定着前: N件
- [x][x][ ][ ] 概要を言える: N件
- [x][x][x][ ] 自分の言葉で説明できる: N件

### 基礎トピックの進捗
カバー済み: N / 全体M件

カテゴリ別:
  Web基礎         N/5
  SQL・DB         N/5
  API設計         N/3
  ...

### 次に学ぶといいトピック
1. {トピック名}（{理由}）
2. {トピック名}（{理由}）
3. {トピック名}（{理由}）
```

期間内の活動が0件の場合は「この期間のスクラップ活動はありません」と表示してレポートを終了する。

### 7. レポートをファイルに保存する

以下のパスに保存する:

```
_reports/{YYYY-MM-DD}.md
```

ファイルの内容はレポートの表示内容をそのまま書き出す。
`.gitignore` により git 追跡対象外となる。

### 8. 学習トピックを選んでもらう

レポート表示・保存の直後に以下を表示する:

```
「次に学ぶといいトピック」から1つ選ぶと、スクラップサイズの質問を3つ出します。
番号（1〜3）を入力してください。スキップする場合はそのままEnterを押してください。
```

ユーザーがEnterのみ・またはスキップと回答した場合はそこで終了する。

### 9. 選んだトピックをスクラップ粒度に分解して出題

選ばれたトピックを、独立して保存できるスクラップサイズの問いに3つ分解する。

分解の基準:
- 各問いは1つの概念・1つの問いに絞る（CLAUDE.md の Q 形式と同じ粒度）
- 互いに依存せず、単独でスクラップとして成立する内容にする
- 「〜とは何か」「〜と〜の違いは」「〜はなぜ必要か」など核心を問う1文にする

以下の形式で出題する:

```
以下の3問、順番に答えてみてください。わかる範囲でOKです。

Q1. {問い}
Q2. {問い}
Q3. {問い}
```

ユーザーが答えたら、Q1 から順に通常のスクラップ保存フロー（CLAUDE.md「スクラップを保存する」手順の4〜8）を実行する。
- A を開示してフィードバック
- 理解度（1〜4）を確認
- ファイル保存 + `_index.md` 更新 + git commit/push

3問すべて完了したら終了する。
