# notebooklm-scrap

全スクラップ（アーカイブ含む）を読み込み、意味的にグルーピングして NotebookLM 用の音声読み上げ原稿を生成する。

## 手順

1. `_index.md` を読み込み、全スクラップのファイルパスを取得する
2. `_archive/` 配下のスクラップも Glob で検索して追加する
3. 全スクラップファイルを読み込む
4. スクラップを意味的にグルーピングしてトピックを決める:
   - スクラップのカテゴリ（SQL/Java/Web等）に縛られなくてよい
   - 複数スクラップをまたいで「音声として話せるひとまとまり」になるよう編集判断する
   - 例：「JavaのnullとSpringのバリデーション」「SQLウィンドウ関数の使いどころ」など
   - 1トピック = 読み上げ約3分（500〜600字程度）を目安にする
5. トピックごとに `_notebooklm/{トピック名}.md` を生成・上書きする
6. 生成したファイル一覧をユーザーに報告する

## 原稿のフォーマット

```markdown
# {トピックタイトル}

{本文 — プレゼン口調・やや砕け目の文体で500〜600字程度}

---
_対象スクラップ: {ファイルパス1}, {ファイルパス2}, ..._
```

## 原稿の文体ルール

- **プレゼン口調・やや砕け目**: 「〜なんですよね」「ここがポイントで」「つまりどういうことかというと」など自然な話し言葉
- 読み上げたときに耳で聞いてわかるよう、記号・コードブロック・箇条書きは使わない
- Q&A 形式にせず、流れのある説明文にする
- 専門用語は初出時に一言補足を入れる

## グルーピングの指針

- 同じ概念の別側面（例：`int-Integer-null.md` と `静的メソッドとインスタンスメソッド-null安全性.md` → 「Javaとnull」）
- 対比・比較で理解が深まるもの（例：SSRとCSRとRSC）
- 問題→解決策の流れで語れるもの（例：正規化の問題点 → 非正規化で解決する場面）
- 無理にまとめず、1スクラップで1トピックにしてもよい

## 再実行時の動作

`_notebooklm/` 配下のファイルをすべて上書き再生成する。スクラップが追加・更新されていれば自動で反映される。
