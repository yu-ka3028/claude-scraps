`_index.md` を読み込んで復習対象を絞り込み、復習セッションを開始する。

## 手順

1. `_index.md` を読み込む
2. **次回復習日 ≤ 本日** のスクラップを抽出する（期日超過分も含む）
3. 抽出件数を確認する:
   - **0件**: 「今日の復習対象はありません」と返して終了する
   - **1〜3件**: そのまま全件出題する
   - **4件以上**: 次回復習日の**昇順**（古い順）で上位3件を選ぶ。残りは「{n}件を持ち越しました」と通知する

4. 選んだスクラップのファイルをそれぞれ読み込む
5. **1問ずつ**以下のサイクルで進める（複数問まとめて出さない）:

```
Claude: 「Q1. {質問文}」
   ↓
ユーザー: 回答
   ↓
Claude: 「正解は〜 / あなたの回答は〜」とフィードバック
       「理解度を教えてください（[x][ ][ ][ ] / [x][x][ ][ ] / [x][x][x][ ] / [x][x][x][x]）」
   ↓
ユーザー: 理解度を申告
   ↓
Claude: スクラップファイルと _index.md を更新して次の問いへ（[x][x][x][x]ならアーカイブ移動）
```

6. 全問終了後、本日の復習サマリーを表示する:

```
3問完了。明日の復習対象: 5件（うち持ち越し 3件）
```

## 理解度更新のルール

更新時に書き換える項目（**スクラップファイルと `_index.md` の両方**を更新）:
- 理解度インジケーター
- 最終復習日 → 本日の日付
- 次回復習日 → 更新後のレベルに対応する日数を本日に加算

| 更新後レベル | 次回復習日 |
|-------------|-----------|
| [x][ ][ ][ ] | 本日 + 1日 |
| [x][x][ ][ ] | 本日 + 3日 |
| [x][x][x][ ] | 本日 + 7日 |
| [x][x][x][x] | アーカイブへ移動 |

[x][x][x][x] の場合:
1. スクラップファイルを `_archive/{カテゴリ}/` へ移動する（`git mv`）
2. `_index.md` から該当行を削除する

## Git コミット・プッシュ

理解度を更新したら、ファイル更新の直後に自動でコミット・プッシュする（確認不要）:

```bash
git add {更新したファイル} _index.md
git commit -m "scrap: review {カテゴリ名} — update N items (YYYY-MM-DD)"
git push origin main
```
