**Q:** 経路列挙で先祖と子孫を辿るSQLの違いは？

**A:** LIKE の主語と述語が逆になる。

```sql
-- 子孫を辿る（/a/b/ 以下を全て取得）
WHERE path LIKE '/a/b/' || '%'
-- テーブルの path カラム が主語、固定値にワイルドカードを付与

-- 先祖を辿る（/a/b/c/d/ の親パスを全て取得）
WHERE '/a/b/c/d/' LIKE path || '%'
-- 固定値 が主語、テーブルの path カラムにワイルドカードを付与
```

| 方向 | LIKEの主語 | パターン |
|------|-----------|---------|
| 子孫 | `path`（カラム） | `現在のpath \|\| '%'` |
| 先祖 | `現在のpath`（固定値） | `path \|\| '%'`（カラムに `%` を付与） |

**メモ:** PostgreSQL は `||` で文字列結合できるが、MySQL 系は `||` が OR 演算子のため `CONCAT(path, '%')` が必要。
