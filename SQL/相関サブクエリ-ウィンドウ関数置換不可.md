**Q:** 相関サブクエリをウィンドウ関数に置き換えられないのはどんなとき？

**A:** サブクエリのフィルター条件に**外側クエリのテーブルの列**（例: `r.date`）が使われているとき。ウィンドウ関数は JOIN より前に計算されるため、その時点で外側テーブルの列値を参照できない。

**メモ:**
- `テーブルエイリアス.列名` の形式 → 外側テーブルの列 → 行ごとに変わると判断できる
- 集合の境界が**定数** → 事前に WHERE で絞れる → ウィンドウ関数 OK
- 集合の境界が**外側テーブルの列** → 行ごとに変わる → ウィンドウ関数では表現できない
- SQL の処理順序: `FROM/JOIN → WHERE → SELECT（ウィンドウ関数） → ORDER BY`
- `OVER()` の中に WHERE は書けない（フィルタリングとウィンドウ定義は役割が異なるため）
