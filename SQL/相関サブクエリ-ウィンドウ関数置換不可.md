**Q:** 相関サブクエリをウィンドウ関数に置き換えられないのはどんなとき？

**A:** サブクエリのフィルター条件に**外側クエリのテーブルの列**（例: `r.date`）が使われているとき。ウィンドウ関数は JOIN より前に計算されるため、その時点で外側テーブルの列値を参照できない。

**メモ:** ウィンドウ関数の基本は [[ウィンドウ関数-OVER句]] を参照。
- `テーブルエイリアス.列名` の形式 → 外側テーブルの列 → 行ごとに変わると判断できる
- 集合の境界が**定数** → 事前に WHERE で絞れる → ウィンドウ関数 OK
- 集合の境界が**外側テーブルの列** → 行ごとに変わる → ウィンドウ関数では表現できない
- SQL の処理順序: `FROM/JOIN → WHERE → SELECT（ウィンドウ関数） → ORDER BY`
- `OVER()` の中に WHERE は書けない（フィルタリングとウィンドウ定義は役割が異なるため）
- OVER() のスポットライトは FROM/JOIN の結果セットのみ対象。他テーブルを動的に参照する構文がない
- WHERE と OVER() の違い: WHERE は行を削除する操作、OVER() は各行に「計算に使う範囲（スポットライト）」を与える操作
- JOIN して同じレベルに WHERE を置くと事前にフィルタされ OVER() はその後に動く → 置き換え可能になる場合がある
- LEFT JOIN + WHERE（右テーブル列条件）は NULL 行が消えて LEFT JOIN の意味が壊れる
- LEFT JOIN の場合は ON 句に条件を移すと NULL 行は保持できるが、OVER() は全行を見るため境界問題は解決しない
- **動的な境界 + LEFT JOIN の組み合わせ → 相関サブクエリが素直な解**
